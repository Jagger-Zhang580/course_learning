# AI Agent 框架系统学习课程 (6-8 周)

## 课程目标
掌握 AI Agent 核心概念、主流框架实现及工程化最佳实践，能够独立设计与开发具备工具使用、记忆管理、多智能体协作能力的实际应用。

## 课程大纲

### 第1周：AI Agent 基础与生态探索
- **核心概念**：Agent vs 传统LLM应用；感知-决策-行动循环；自主性与目标导向
- **主流框架横评**：LangChain（生态最完整）、LlamaIndex（数据聚焦）、AutoGPT（早期探索者）、MetaGPT（多Agent协作先行者）
- **环境搭建**：Python 虚拟环境、API 密钥管理（OpenAI/开源替代方案）、依赖版本锁定
- **Demo**：构建基于 LLM 的基本 Chain（提问-回答机器人）
  - 实现：使用 `LangChain` + `OpenAI` 完成简单问答
  - 代码路径：`week1/basic_chain.py`
- **作业**：阅读《Towards Agents》一文；尝试更换不同 LLM 提供商

### 第2周：Agent 核心范式与工具使用
- **深度解析**：ReAct (Reasoning + Acting) 框架；思考-行动-观察循环；工具抽象与封装
- **工具设计原则**：幂等性、错误处理、结果格式化；常用工具模板（搜索、计算、API调用）
- **实战**：集成网络搜索工具（DuckDuckGo/SerpAPI）到 Agent
  - 实现：让 Agent 能自主决定何时使用搜索工具回答问题
  - 代码路径：`week2/react_agent_with_search.py`
- **作业**：添加一个自定义工具（如天气查询或单位转换）

### 第3周：记忆系统设计与实现
- **记忆分类**：短期记忆（对话窗口/缓冲区）、长期记忆（向量数据库/知识图谱）、情景记忆（经验库）
- **技术选型**：FAISS  vs ChromaDB vs Pinecone；嵌入模型选择；相似度搜索优化
- **实战**：为 Agent 添加长期记忆能力（记住用户偏好和历史事实）
  - 实现：使用 `ChromaDB` 存储对话片段；通过相似度检索增强上下文
  - 代码路径：`week3/memory_enhanced_agent.py`
- **作业**：实现一个“偏好学习”功能（Agent 能记住用户喜欢的回复风格）

### 第4周：高级工具链与 API 编排
- **工具链设计**：工具的组合使用（工具 A 的输出作为工具 B 的输入）；异步执行与并行调用
- **函数调用范式**：OpenAI 函数调用；JSON Schema 定义工具接口；类型安全考量
- **实战**：构建多工具协作的 Agent（例如：查询股票数据 → 分析趋势 → 生成报告）
  - 实现：集成财经 API + 数据分析工具 + 报告生成模板
  - 代码路径：`week4/multi_tool_agent.py`
- **作业**：添加错误重试机制和超时控制

### 第5周：多智能体协作与工作流编排
- **协作模式**：顺序流水线（Assembly Line）；并行处理（Map-Reduise）；投票/仲裁机制；角色扮演（Role-Play）
- **框架对比**：MetaGPT（标准化角色与流程）；ChatDev（软件开发全流程模拟）；AgentVerse（通用协作平台）
- **实战**：开发双Agent 协作系统（研究员 + 撰写员）
  - 实现：研究员负责信息收集和初步分析；撰写员负责生成结构化报告
  - 代码路径：`week5/collaborative_agents.py`
- **作业**：尝试添加第三个角色（如审核员或美工设计师）

### 第6周：安全、评估与生产化考量
- **安全防护**：提示词注入防范；工具调用权限控制；输出内容过滤
- **成本与性能**：Token 使用监控；缓存策略；并发度控制；模型选择策略（大模型 vs 小模型+微调）
- **评估方法**：任务成功率；步骤效率；人工偏好对齐（简单版 RLHF 概念）；A/B 测试思路
- **实战**：为之前的 Agent 添加安全防护层和成本追踪
  - 实现：输入审查（关键词过滤）；调用计数器；结果安全检查
  - 代码路径：`week6/secure_agent.py`
- **作业**：设计一个简单的 Agent 性能基准测试

### 第7-8周：Capstone 项目（实战演练）
- **项目选项**（任选其一，鼓励创新）：
  1. **智能客服助手**：集成企业 FAQ 知识库 + 工单系统 + 情感分析
  2. **个人生活管家**：日程管理 + 邮件摘要 + 提醒服务 + 天气建议
  3. **数据分析助手**：自动读取数据 → 生成洞察 → 制作可视化建议 → 写报告
- **里程碑**：
  - 第7周初：提交项目提案（功能清单 + 技术方案 + 风险点）
  - 第7周末：完成核心功能演示（基本交互流程可运行）
  - 第8周中：完成次要功能与用户界面（CLI 或简单 Web 界面）
  - 第8周末：项目展示与互评（每人 5 分钟演讲 + 2 分钟 Q&A）
- **支持提供**：
  - 项目模板仓库（含基础框架和常见工具）
  - 答疑时间（每周固定 2 小时办公时间）
  - 评估标准表（功能完整性 40% + 代码质量 30% + 创新性 20% + 演示效果 10%）

## 环境与准备
- **硬件要求**：普通笔记本即可（建议 8GB+ RAM）
- **软件要求**：Python 3.9+；Git；VS Code 或 PyCharm
- **准备工作**：
  1. 注册 OpenAI 账号获取 API 密钥（或准备使用本地开源模型）
  2. 安装 Docker（可选，用于部分工具的隔离运行）
  3. 预习基本 Python 和 REST API 概念

## 课程特色
- **理论与实践平衡**：每周 1.5 小时理论 + 2.5 小时实验
- **递进式学习**：从单个组件到复杂系统，循环加深理解
- **产出导向**：每周都有可运行的代码 Demo，最终交付可展示的项目
- **就业导向**：涵盖企业级 Agent 开发的全流程经验（设计→实现→测试→部署）
- **可迁移性**：强调框架思想而非具体 API 实现，所学知识易于迁移到新技术栈

> 备注：BERT/Transformers 等深度学习内容在此课程中作为选补材料（第6周可选），核心重点在 Agent 架构而非底层模型训练。如需深入模型细节，可另行学习 NLP 专项课程。
